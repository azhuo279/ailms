# Canonical UI Component List for an AI Enabled Logistics Management Web App

## Framing assumptions

For a 72-hour design brief, the most useful outcome is not an exhaustive catalogue of every conceivable widget. It is a **normalized, implementation-ready component inventory** that covers the patterns most commonly repeated in enterprise web applications: dense data views, search and filtering, transactional forms, navigation scaffolds, layered interactions, and feedback surfaces. That emphasis is consistent with the way major systems such as Material, Fluent, Carbon, Atlassian Design System, and SAP Fiori structure their component libraries and guidance for enterprise software. The canonical names below intentionally **normalize vendor-specific naming** so your design file, codebase, and QA language stay consistent. For example, this report uses **Tag** as the canonical name for removable value tokens even though some systems call related patterns *chips* or *lozenges*; it uses **Message Bar** for persistent inline or banner-style notices and **Toast** for transient updates; and it uses **Combobox** only when typing and suggestion selection are both part of the interaction, reserving **Select** for form submission from a predefined list. That normalization is a synthesis of cross-system usage guidance, not a verbatim copy of any one vendor.
